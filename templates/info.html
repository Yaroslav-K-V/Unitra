Product overview

Overview

Unitra is a local-first Python testing workspace with three ways to work: Quick for one-off drafts, Workspace for managed repository flows, and `unitra` for automation-friendly terminal use.

What Unitra is

Unitra helps you move from raw Python code to runnable tests without jumping between disconnected tools. It keeps the common path simple, but still gives you a repo-aware workflow when the work stops being a one-file experiment.

Product surface

Quick is the shortest path from pasted code to a local draft test module.
Workspace is the repository view for managed files, previews, reruns, and failure repair.
CLI exposes the same ideas in a scriptable form for CI and power-user flows.
Console gives a Textual terminal surface over the shared core workflows.

How to use it

Start in Quick when you want instant drafts for snippets, scratch files, or small local checks.
Move to Workspace when you need repo context, managed test paths, review before write, or repeated runs.
Use `unitra` when you want JSON output, automation, or repeatable terminal-first workflows.
Open Console when you want the shared workflow in an interactive terminal UI instead of the desktop shell.

Local-first guarantees

Workspace metadata lives locally in .unitra inside the repository.
Settings and API configuration stay on your machine in .env.
Generated tests, previews, run history, and agent profile choices are designed around local repo state first.
AI fallback matters only when it is explicitly configured and the current flow actually needs it.

Best fit

Solo Python work where you want faster test scaffolding without hiding the file-level result.
Repository flows where previewing managed writes is more important than “generate and hope”.
Terminal and CI use cases where structured output matters as much as desktop convenience.

Mental model

Quick: “I have code and I need a draft now.”
Workspace: “I have a repo and I want a calmer review-and-run loop.”
CLI / Console: “I want the same workflow, but in automation or terminal form.”
